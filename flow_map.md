# 스파이스 흐름 지도 (Spice Flow Map)

서울 상권 불균형 문제 해결을 위한 인구·소비 흐름 인터랙티브 시각화 플랫폼

> 보완판
> 대상: 2026 서울시 빅데이터 활용 경진대회 시각화 부문
> 세계관: Dune의 "Spice Flow"를 서울 상권 문제에 맞게 재해석

## 0. 보완 진단 요약

### 이 기획서의 강점

- 공공문제가 분명하다. `젠트리피케이션`과 `침체 골목상권 공동화`를 동시에 다룬다.
- 데이터 결합 구조가 좋다. `OD 이동`, `생활인구`, `매출`, `개폐업`, `상권변화지표`, `임대료`가 한 방향으로 연결된다.
- 시각화 콘셉트가 강하다. "어디가 사람과 소비를 빨아들이고, 어디가 말라가는가"가 한 장면으로 설명된다.
- 정책 연결성이 있다. 단순 대시보드가 아니라 `예산 우선순위`, `보호구역`, `접근성 개선`까지 이어진다.

### 심사에서 바로 지적받을 가능성이 큰 부분

1. `OD 이동량 = 소비 흐름`이라는 가정이 아직 검증되지 않았다.
2. 데이터 단위가 다르다. `시간별`, `월별`, `분기별` 데이터를 어떻게 정렬하는지가 약하다.
3. 성과지표가 부족하다. "좋아 보인다"가 아니라 `예측력`, `정책 활용성`, `사용성`을 수치로 제시해야 한다.
4. AI 제언 부분은 아직 설명형에 가깝다. `근거 데이터`, `출력 형식`, `환각 방지` 규칙이 필요하다.
5. 정책 적용 경로가 추상적이다. `누가`, `언제`, `어떤 문서`로 활용하는지 운영 흐름이 있어야 한다.
6. 개인정보·편향·오판 비용에 대한 설명이 없다.

### 이번 보완본에서 추가한 핵심

- `OD를 소비 대리변수로 쓰는 검증 설계`
- `정량 KPI`와 `베이스라인 비교`
- `데이터 한계·윤리·편향 통제`
- `정책 운영 프로세스`와 `담당 부서 연결`
- `리스크 대응`과 `MVP 축소안`

## 1. 해결하려는 공공문제

서울시 소상공인의 현실은 두 가지 극단으로 갈린다.

- 과열 상권: 성수, 홍대, 한남, 명동 등 일부 지역으로 유동인구와 소비가 집중되며 임대료가 상승하고 기존 소상공인이 밀려난다.
- 침체 상권: 다수의 골목상권은 유입 흐름이 약해 폐업이 반복되고 상권 전체가 공동화된다.

이 문제의 핵심은 "지금 어느 상권이 좋은가"가 아니라 "사람과 소비가 왜 특정 상권으로만 흐르는가"이다.

기존 서비스는 개별 상권의 정적 상태를 보여주는 데 강하지만, 상권 간 이동 흐름의 `방향`, `흡수`, `방출`, `단절`을 보여주지 못한다.

## 2. 프로젝트 한 줄 정의


### 핵심 질문

- 어느 상권이 사람과 소비를 빨아들이는가
- 어느 상권이 지속적으로 방출형으로 바뀌는가
- 어느 구간에서 흐름이 끊겨 침체가 고착되는가
- 어디에 먼저 보호 정책이나 활성화 예산을 투입해야 하는가

### 기존 서비스와의 차이

| 구분 | 기존 상권 분석 | 스파이스 흐름 지도 |
| --- | --- | --- |
| 분석 단위 | 개별 상권 상태 | 상권 간 흐름 네트워크 |
| 관점 | 정적 스냅샷 | 시계열 변화 + 이동 방향 |
| 핵심 질문 | 이 상권은 어떤가 | 왜 이 상권으로 가지 않나 |
| 정책 연결 | 제한적 | 우선순위 자동 산출 |
| AI 활용 | 낮음 | 근거 기반 정책/창업 인사이트 |

### 4.2 결합 원칙

- 기본 공간 키: `행정동 코드`
- 상권 단위 분석 키: `상권 코드`
- 시간 정렬 원칙:
  - `생활인구`: 시간대 패턴 탐색
  - `OD`: 월 단위 흐름 구조
  - `매출/점포/상권변화`: 분기 단위 결과 변수
- 분석 테이블은 최종적으로 `상권-분기` 단위로 정규화한다.

### 4.3 시간축 정렬 설계

이 프로젝트는 시간 단위가 서로 다르기 때문에, 아래처럼 `분석용 공통 시계열`을 정의해야 설득력이 생긴다.

| 데이터 | 원 단위 | 분석용 변환 |
| --- | --- | --- |
| 생활인구 | 시간 | 출근/점심/저녁/주말 피크 지표 |
| OD 이동 | 월 | 월별 순유입, 목적별 순유입, 3개월 롤링 |
| 매출 | 분기 | 분기 결과값 |
| 폐업률/점포수 | 분기 | 분기 결과값 |
| 임대료 | 분기 | 분기 변화율 |

권장 방식:

- 설명 변수는 `t분기 이전 3개월 평균`으로 집계한다.
- 결과 변수는 `t분기 매출`, `t+1분기 폐업률`, `t+1분기 GRI 악화`로 둔다.
- 이렇게 해야 "흐름이 먼저 바뀌고, 상권 상태가 나중에 반응한다"는 서사가 성립한다.

### 4.4 공간 결합 설계

현재의 `행정동 중심점 -> 반경 500m 내 상권` 방식은 빠르지만 오차가 생길 수 있다. 보완안은 다음과 같다.

1. 1순위: `행정동 폴리곤`과 `상권 폴리곤`의 면적 교차 비율로 가중 배분
2. 2순위: 폴리곤이 없으면 `centroid + nearest k 상권`에 거리 역수 가중치 적용
3. 3순위: 상권이 과도하게 큰 경우 `버퍼 반경`을 자치구별로 차등 적용

이 보완을 넣으면 심사에서 나오는 "왜 500m인가?" 질문에 답할 수 있다.

### 4.5 가장 중요한 보완: OD는 소비의 대리변수라는 검증

심사위원이 가장 먼저 물을 질문은 이것이다.

> "사람 이동량이 늘었다고 실제 소비가 늘었다고 볼 수 있나?"

이 질문에 답하기 위해 문서에 아래 검증 설계를 반드시 넣어야 한다.

#### 검증 가설

- `H1`: 순유입 증가율이 높은 상권은 같은 분기 또는 다음 분기 매출 증가율이 높다.
- `H2`: 흐름 단절 점수가 높은 상권은 다음 분기 폐업률 상승 가능성이 높다.
- `H3`: 순유입 급증 + 임대료 상승 + 프랜차이즈 비중 상승이 함께 나타나는 상권은 젠트리피케이션 위험이 높다.

#### 검증 방법

| 항목 | 방법 |
| --- | --- |
| 매출 검증 | `순유입`, `목적별 유입`, `시간대 피크`와 분기 매출 증가율의 상관 분석 |
| 폐업 검증 | 침체 예측 상권과 실제 폐업률 상승 상권의 정합도 비교 |
| GRI 검증 | 위험 상권과 다음 분기 임대료 상승/프랜차이즈화의 일치도 측정 |

#### 제시할 KPI

| 분류 | KPI 예시 |
| --- | --- |
| 설명력 | `순유입-매출 증감 상관계수` |
| 조기 경보 | `폐업 위험 상위 20개 상권 적중률` |
| 비교 우위 | `상권변화지표 단독 대비 개선율` |
| 운영성 | `정책 검토 회의용 보고서 자동 생성 시간` |

#### 베이스라인 비교

- 베이스라인 1: 기존 `상권변화지표`만 사용
- 베이스라인 2: `생활인구`만 사용
- 베이스라인 3: `매출/폐업`의 직전 추세만 사용
- 제안 모델: `OD + 생활인구 + 매출 + 점포 + 임대료 + 상권변화지표`

이 비교가 있어야 "기존 방법으로도 되지 않나?"라는 반론을 막을 수 있다.

### 4.6 데이터 한계와 윤리

- 이 시스템은 `개인 이동 추적`이 아니라 `집계 데이터`를 다룬다.
- OD 이동량은 `실제 결제 데이터`가 아니라 `소비 가능성의 선행 지표`다. 따라서 정책 제언은 `근거 보조`로 사용하고 자동 집행 근거로 쓰지 않는다.
- 특정 상권을 "위험"으로 표시할 때 낙인 효과가 생길 수 있으므로 공개 화면에는 등급과 설명 문구를 분리한다.
- 자치구별 데이터 편차가 존재할 수 있으므로 구 단위 정규화와 이상치 검토가 필요하다.

## 5. 핵심 분석 모듈

### 5.1 Module A: 상권 흐름 네트워크

상권을 노드, 상권 간 이동을 방향성 있는 엣지로 둔 `유향 그래프`를 구축한다.

핵심 지표:

- `in_flow`: 유입량
- `out_flow`: 유출량
- `net_flow = in_flow - out_flow`
- `flow_share`: 자치구 내 점유율
- `centrality`: 네트워크 중심성

권장 분류 기준:

| 유형 | 조건 |
| --- | --- |
| 흡수형_과열 | 순유입 상위 + 임대료 상승 + 프랜차이즈 비중 상승 |
| 흡수형_성장 | 순유입 증가 + 매출 증가 + 임대료 안정 |
| 방출형_침체 | 순유출 지속 + 폐업률 상승 |
| 고립형_단절 | 유입/유출 모두 매우 낮고 주변 연결도 약함 |
| 안정형 | 변동성이 낮고 지표가 균형적 |

보완 포인트:

- 절대값 기준 대신 `자치구 내 백분위`를 함께 써야 지역 규모 차이를 줄일 수 있다.
- `주거지 성격이 강한 상권`은 귀가 흐름 때문에 방출형으로 오판될 수 있으므로 목적별 분해가 필요하다.

### 5.2 Module B: 젠트리피케이션 위험 지수

권장 개념식:

```text
GRI =
0.30 * 순유입 증가율 +
0.25 * 임대료 상승률 +
0.20 * 프랜차이즈 비중 증가 +
0.15 * 매출 급증률 -
0.10 * 독립 소상공인 비중
```

보완 포인트:

- `분기별 z-score`나 `구 단위 min-max`로 표준화해 지역별 규모 편차를 줄인다.
- 상권 유형별로 임계값을 다르게 둘 수 있다. 예를 들어 관광 상권은 기본 변동성이 크다.
- 위험 지수는 최종 판단이 아니라 `모니터링 트리거`로 위치시켜야 한다.

### 5.3 Module C: 흐름 단절 구간 탐지

기존의 단순 최단경로보다 아래 방식이 실무적으로 더 낫다.

1. 대상 상권 주변 1차 상권의 유입 분해
2. 대상 상권으로 넘어오지 않는 상류 흐름 확인
3. 보행 접근, 교통 결절, 야간 체류 부족, 목적 편향 여부를 함께 기록

권장 출력:

```json
{
  "상권명": "신림 골목상권",
  "단절유형": "주말 쇼핑 유입 부족",
  "상류차단지점": ["서울대입구역 상권", "봉천역 상권"],
  "정책후보": ["보행 유도 사인", "야간 프로그램", "생활권 마케팅"]
}
```

### 5.4 Module D: AI 정책 제언

AI는 `설명 생성기`가 아니라 `근거 요약기`로 정의하는 편이 안전하다.

반드시 넣어야 할 규칙:

- 입력은 수치형 지표와 근거 데이터만 사용
- 출력은 `현황`, `근거`, `권고 3개`, `담당 부서`, `기대 지표`의 고정 형식
- 근거 없는 추정 금지
- 데이터 부족 시 "판단 보류" 응답 허용

권장 출력 형식:

```text
현황: 방출형_침체 상권이며 최근 2개 분기 순유출이 확대됨.
근거: 순유출 -4,200명/일, 폐업률 17.8%, 중심성 하락.
권고1: 골목 유입 동선 개선 검토.
권고2: 주말 체류형 프로그램 연계.
권고3: 위기 소상공인 컨설팅 우선 배정.
담당: 자영업지원과, 지역상권팀, 자치구 경제과.
```

### 5.5 Module E: 정책 우선순위 점수

정책 우선순위는 아래처럼 `위기 정도`와 `개입 가능성`을 함께 봐야 한다.

```text
Priority Score =
0.35 * 순유출 위험 +
0.25 * 폐업 위험 +
0.15 * 연결 단절도 +
0.15 * 생활권 수요 대비 격차 +
0.10 * GRI
```

분류 예시:

- `80+`: 즉시 개입
- `60~79`: 연내 지원
- `40~59`: 모니터링 및 시범사업

## 6. 시각화 설계

### 6.1 핵심 화면

- 메인 지도: 상권 노드 + OD 곡선 + 시간대 애니메이션
- 우측 패널: 상권 상세, GRI, 폐업률, 정책 제언
- 하단 타임라인: 분기/시간 전환
- 상단 필터: 목적, 시간대, 요일, 자치구

### 6.2 시각화 원칙

- 색은 `상권 상태`, 선은 `흐름 방향`, 노드 크기는 `순유입 규모`를 의미한다.
- 침체 상권은 시선이 가되 과장되지 않도록 펄스 강도를 낮춘다.
- 실시간처럼 보여도 실제로는 `최신 집계 기준`이라는 설명을 붙인다.
- 상권 클릭 시 "왜 이렇게 분류됐는가"가 바로 보여야 한다.

### 6.3 3분 발표용 시연 시나리오

1. 오전 8시 출근 흐름으로 강남·여의도 흡수형 패턴 제시
2. 오후 2시 쇼핑 흐름으로 홍대·성수 집중 제시
3. 침체 상권 클릭 후 단절 구간과 정책 제언 표시
4. 분기 전환으로 같은 상권의 악화/회복 추세 비교

## 7. 정책 연결과 운영 프로세스

### 7.1 실제 활용 흐름

| 단계 | 산출물 | 활용 주체 |
| --- | --- | --- |
| 월간 데이터 적재 | 상권 흐름 요약표 | 데이터 운영 담당 |
| 분기 분석 | 위험 상권 목록, 우선순위 점수 | 서울시 정책팀 |
| 현장 검토 | 자치구 의견, 현장 사진, 민원 | 자치구 경제과 |
| 지원 결정 | 활성화/보호/접근성 개선안 | 본청 + 자치구 |
| 사후 평가 | 다음 분기 변화 비교 | 정책평가 담당 |

### 7.2 담당 부서 연결 예시

- `흡수형_과열`: 임차인 보호, 상생협약, 모니터링 강화
- `방출형_침체`: 골목상권 활성화 사업, 컨설팅, 상권 프로그램
- `고립형_단절`: 보행 환경, 이정표, 교통 접근 보완
- `흡수형_성장`: 보호구역 검토, 임대료 관찰, 선제 관리

### 7.3 운영상 중요한 보완

- 화면만 보여주면 부족하다. `회의용 1장 리포트 PDF` 자동 생성 기능을 넣는 편이 실무 활용성이 높다.
- AI 문장은 참고용이고, 최종 정책안은 수치표와 함께 저장되어야 한다.

## 13. 심사 대응용 한 줄 메시지

- 이 서비스는 "유명 상권이 왜 뜨는가"보다 "왜 어떤 골목은 계속 비는가"를 설명한다.
- 기존 상권 서비스가 `상태판`이라면, 스파이스 흐름 지도는 `흐름 지도`다.
- 핵심 가치는 예쁜 지도 자체가 아니라 `선행 신호 기반 정책 판단`이다.

## 부록 A. 발표에서 바로 쓸 문장

> 서울의 문제는 상권이 없는 것이 아니라, 흐름이 특정 상권에만 과도하게 집중된다는 점입니다.  
> 스파이스 흐름 지도는 이 이동의 방향과 단절 지점을 데이터로 시각화해, 침체 상권을 사후 대응이 아니라 선제 대응 대상으로 바꿉니다.

## 부록 B. 문서에서 특히 강조할 보완 문장

- 본 서비스는 이동량을 소비의 직접 측정치가 아니라 `선행 대리 지표`로 사용하며, 분기 매출·폐업 데이터와의 정합성 검증을 함께 제시한다.
- 정책 제언은 자동 의사결정이 아니라 `근거 요약과 우선순위 추천`으로 한정한다.
- 시범 단계에서는 1~2개 자치구를 대상으로 검증 후 전역 확장한다.
