# spiceMap 서비스 최종 기능 개요

> 2026 서울시 빅데이터 활용 경진대회 (창업 부문) 제출작
> 작성일: 2026-05-12

## 서비스 한 줄 정의

> **"왜 이 상권이 침체됐는가 — 흐름으로 보는 서울 상권 위험 지도 + 창업 입지 추천"**

기존 상권 분석 도구는 매출·폐업률 같은 결과 스냅샷만 보여준다.  
spiceMap은 인구 이동 흐름을 원인으로 분석하고, 거기서 위험 신호를 감지하며, 창업자에게는 입지 추천까지 제공한다.

---

## 핵심 기능 5가지 (일반 사용자 기준)

### 1. OD 이동 흐름 시각화

서울시 수도권 광역 OD 데이터(OA-22300)를 바탕으로 행정동 → 상권 단위 인구 이동을 지도 위 흐름 애니메이션으로 표시한다.

- 목적별(출근·귀가·쇼핑·여가) 필터링
- 시간대별(0~23시) 슬라이더로 시간축 재생
- 상위 N개 흐름만 표시하는 강도 조절

### 2. 젠트리피케이션 위험지수 GRI

폐업률(OA-15577) · 순유출(OD NetworkX) · 네트워크 고립도 3가지를 가중 결합해 상권별 0~100 점수로 표시한다.

- **H3 검증 결과**: GRI 상위 20% 상권의 폐업률이 하위 80%의 **14.5배** (p=8×10⁻³¹, n=1,650)
- 분기별 추이 차트 + 자치구 내 백분위 순위 병행 표시

### 3. AI 창업 어드바이저

창업자가 **업종 + 관심 자치구**를 선택하면, 백엔드가 다음을 수행한다.

```
① GRI · 유동인구 · 폐업률 · 점포 수 · 네트워크 중심성 5개 지표를 가중 합산
② 전체 상권을 점수 순 정렬 후 추천(상위 30%) / 주의(중간 40%) / 비추천(하위 30%) 3등급 분류
③ 각 등급에서 3개씩, 총 9개 선별
④ Claude Haiku API가 9개 상권의 데이터를 읽고 자연어 해설 JSON 생성
⑤ 프론트엔드에 추천·주의·비추천 카드 형태로 표시
```

기존 서비스와 다른 점: 추천만 보여주는 것이 아니라 **"왜 비추천인지"를 수치 근거와 함께** 보여준다.

**어드바이저 점수 산정 가중치**

| 지표 | 가중치 | 방향 |
|------|--------|------|
| GRI (위험도) | 0.25 | 낮을수록 유리 |
| 폐업률 | 0.25 | 낮을수록 유리 |
| 유동인구 (OD 순유입) | 0.20 | 높을수록 유리 |
| 점포 수 (log 변환) | 0.20 | 중간값 선호 (경쟁 포화 반영) |
| 네트워크 중심성 | 0.10 | 높을수록 유리 |

### 4. 자동 정책 추천 카드

상권의 GRI 신호에 따라 공무원이 바로 결재 올릴 수 있는 정책 카드를 규칙 엔진으로 즉시 생성한다.

| 룰 | 대상 상황 |
|----|----------|
| R4 | 임대료 급등 → 임대료 안정 정책 |
| R5 | 프랜차이즈 잠식 → 진입 관리 정책 |
| R6 | 유동인구 감소 → 공공 유동 유도 정책 |
| R7 | 폐업 임박 → 긴급 대응 정책 |

생성형 AI 없이 수치 기반으로 동작하므로 환각이 없고, CSV 다운로드로 결재 문서화가 가능하다.

### 5. 흐름 단절 시각화 (실 도로 경로)

ORS(OpenRouteService) API로 출발-도착 좌표 사이 실제 도로 경로를 받아, 직선이 아닌 **실 도로 위로 인구 흐름 입자 애니메이션**을 구현했다.  
API 장애 시 자동 fallback + 정적 캐시로 발표 중 끊김이 없다.

## 사용자 흐름

```
지도 진입
  → 자치구 선택 (복수 선택 가능)
  → 분기 선택 (2025Q1~Q4)
  → 업종 선택 + 분석하기
      → AI 창업 어드바이저: 추천 / 주의 / 비추천 9개 카드
  → 상권 클릭
      → GRI 게이지 + 정책 카드 + 분기별 추이
  → OD 흐름 ON
      → 시간대별 이동 재생 (목적 필터)
```

## 발표 시연 전용 기능 (일반 사용자 비노출)

URL에 `?hero=1`을 붙이면 시연 모드가 활성화된다. 키보드 `1~4`로 구간별 자동 이동.

| 키 | 화면 |
|----|------|
| `1` | 인트로 — 강남·관악 색 대비 |
| `2` | 신림 골목상권 상세 + R4 정책 카드 |
| `3` | CSV 다운로드 |
| `4` | 검증 보고 — H1·H2·H3·B1·B3 통계 5카드 |

**검증 보고 카드 내용** (키 `4` 진입 시)

| ID | 검증 내용 | 결과 |
|----|----------|------|
| H1 | 순유입↑ → 매출↑ | r=0.106, p=2.83×10⁻⁵, n=1,565 |
| H2 | 흐름단절 → 폐업↑ | r=−0.558, p=2.2×10⁻⁴ (방향 유의, 표본 한계 명시) |
| H3 | GRI 상위 20% → 폐업↑ | gap=1.009%p, 14.5배, p=8×10⁻³¹ |
| B1 | 서울시 공식 지표 대비 | 187개 추가 위험 상권 발굴 |
| B3 | 기존 매출 추세 모델 대비 | 231개 추가 신호 감지 |

## 데이터 처리 규모

| 항목 | 수치 |
|------|------|
| 분석 상권 수 | 강남·관악 약 1,650개 |
| OD 흐름 집계 | 분기 최대 183,000행 |
| 통계 검증 표본 | H1 n=1,565 / H3 n=1,650 |
| 활용 공공 데이터셋 | 5종 (열린데이터광장 4 + 공공데이터포털 1) |
| AI 계층 | 생성형(Claude Haiku) + 그래프 분석(NetworkX) + 통계(scipy) |

## 기술 스택 요약

```
수집 파이프라인 (Python)
  → PostgreSQL + PostGIS + Redis
  → FastAPI (backend/api)
  → React + Vite + MapLibre + Deck.gl (frontend)
  → Anthropic Claude Haiku (AI 창업 어드바이저)
  → NetworkX (그래프 분석)
  → scipy (통계 검증)
  → OpenRouteService (실 도로 경로)
```
