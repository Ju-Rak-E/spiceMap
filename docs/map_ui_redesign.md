# Map UI Redesign Plan

> 작성일: 2026-04-17  
> 최종 갱신: 2026-04-17  
> 대상: `frontend` 지도/분석 패널 UI  
> 기준 레퍼런스: `palantir-for-family-trips`의 정보 구조만 차용

---

## 1. 배경

현재 지도는 Week 2 기준 핵심 시각화 요소를 이미 갖췄다.

- 상권 노드
- OD 흐름 곡선
- 상권 유형 색상/기호 표현
- 우측 제어 패널

다만 화면이 `보이는 것`에 비해 `읽히는 것`은 부족했다.

- 현재 화면이 무엇을 뜻하는지 한눈에 해석하기 어렵다
- 지도와 우측 패널의 역할이 명확히 분리되지 않았다
- 선택 상권의 상세 정보 구조가 준비되어 있지 않았다
- 범례, 상태 배지, 보조 문구의 위계가 느슨했다

이번 재설계는 레퍼런스의 톤을 복제하는 것이 아니라, 아래 원칙만 차용한다.

- 엄격한 정보 위계
- 지도 중심 구조
- 절제된 다크 톤
- 데이터 상태와 경고 표현의 분리

현재 브랜치 `feature/map-ui-redesign`에는 아래 항목이 반영되어 있다.

- 상단 종합 해설바 구현
- 우측 분석 패널 구조 재편
- 상권 유형 드롭업 선택 패널 적용
- 선택 상권 상세 카드 구현
- GRI 이력 미니 차트 연동
- 다크 분석형 시각 토큰 정리

## 2. 디자인 원칙

### 2.1 지도 중심

지도는 계속 메인 뷰다.  
패널은 지도를 보조해야 하며, 지도를 대체하면 안 된다.

### 2.2 패널은 제어판이 아니라 분석 패널

우측 패널은 단순 컨트롤 묶음이 아니라 아래 역할을 가져야 한다.

1. 현재 보기 조건 설명
2. 대표 수치 요약
3. 상권 유형 해석
4. 선택 상권 상세 진입점

### 2.3 색은 의미 전달 전용

- 빨강: 즉시 개입 / 위험
- 주황: 연내 지원 / 주의
- 초록: 안정
- 회색: 단절 / 침체

배경과 카드에는 의미색을 남용하지 않는다.

### 2.4 군사/작전실 톤은 배제

레퍼런스의 `situation room` 느낌은 차용하지 않는다.  
우리 화면은 정책 실무자와 심사위원이 3분 안에 읽을 수 있는 `정책 분석형 대시보드`여야 한다.

## 3. 화면 구조

### 3.1 Header

상단 고정 바는 3구역으로 나눈다.

- 좌측: 제품명
- 중앙: 현재 화면 종합 해설
- 우측: 데이터 상태 / 시범 범위

해설 문장은 아래 요소를 반드시 포함한다.

- 시간대
- 목적
- 상위 N개 흐름 표시 수
- 선택된 상권 유형

### 3.2 Map Surface

- 베이스맵은 더 눌러서 노드와 흐름이 먼저 보이게 한다
- 경계 폴리곤은 존재감만 남기고 전면으로 나오지 않게 한다
- 파티클은 방향 보조 수준으로 유지한다
- 선택된 상권은 hover와 별도로 확실히 식별 가능해야 한다

### 3.3 Right Analysis Panel

패널 구조는 아래 순서를 고정한다.

1. 현재 보기 설정
2. 대표 수치
3. 상권 유형 해석
4. 선택 상권

`가시화 밀도`는 유지하되, `상위 N개 흐름 표시` 문구를 함께 보여 실제 동작을 숨기지 않는다.

### 3.4 Legend

상권 유형 선택 UI는 고정 범례가 아니라 `우하단 드롭업 메뉴`로 둔다.

- 기본 상태는 접힘
- 버튼 라벨은 현재 선택 상태를 요약
  - 예: `전체 유형`, `흡수형 성장`, `3개 유형 선택`
- 클릭 시 위로 펼쳐지는 패널에서 유형을 선택
- 패널 내부 항목은 `색 + 기호 + 이름 + 1줄 설명` 구조 유지
- `전체 선택/전체 해제` 유지
- 바깥 클릭 또는 `Esc` 입력 시 닫힘

즉, 범례는 항상 펼쳐진 설명 패널이 아니라 `필터 겸용 드롭업 선택기`로 동작한다.

### 3.5 Detail Card

선택 상권 상세 카드의 최소 정보는 아래로 둔다.

- 상권명
- 상권 유형
- GRI
- 개입 등급
- 순유입
- 중심성
- GRI 추세

상세 카드는 지도 우측 오버레이로 고정 표시하며, 우측 분석 패널의 `선택 상권` 섹션과 함께 동작한다.

- hover: 빠른 미니 해설
- click: 상세 카드 고정
- close 버튼: 선택 해제

Week 3의 본 상세 패널로 확장 가능한 형태를 유지한다.

## 4. 시각 시스템

### 4.1 중립 팔레트

- 배경: `#0E141B`
- 패널 배경: `#10161D`
- 카드 배경: `#151D26`
- 보더: `#24323F`
- 본문 텍스트: `#E7EEF5`
- 보조 텍스트: `#A6B4C2`
- 묻힘 텍스트: `#6E8093`

### 4.2 강조 규칙

- 제목만 강한 대비
- 대표 수치는 크게
- 보조 설명은 흐리게
- 경고색은 상태 배지와 위험 수치에만 사용

### 4.3 인터랙션 규칙

- hover: 빠른 미니 해설
- click: 상세 카드 고정
- 필터 변경: 상단 해설과 드롭업 라벨 상태를 함께 갱신
- 드롭업 열림 상태에서는 외부 클릭과 `Esc`로 종료

## 6. 검증 기준

### 6.1 첫 화면 이해도

사용자는 5초 안에 아래 3가지를 말할 수 있어야 한다.

- 지금 무엇을 보고 있는가
- 어떤 조건으로 추려진 화면인가
- 어떤 상권이 위험/안정한가

### 6.2 지도 가독성

- 베이스맵보다 노드와 흐름이 먼저 보여야 한다
- 선택 상권과 일반 상권 구분이 명확해야 한다

### 6.3 패널 위계

- 조작과 해석이 섞여 보이지 않아야 한다
- 고정 문구성 카드보다 실제 데이터가 우선해야 한다

### 6.4 품질 기준

- `npm test` 통과
- `npm run build` 통과

### 6.5 현재 검증 결과

- `npm test` 통과
- `npm run build` 통과
- 빌드 시 번들 크기 경고는 존재하지만 배포 실패는 없음

## 7. 결정사항 요약

- 상권 영역 투명도는 사용자 UI에서 제거하고 내부 상수로 유지
- `가시화 밀도`는 유지하되 `상위 N개 흐름 표시`를 병기
- 상단 종합 해설바는 유지
- 상권 유형 선택은 우하단 드롭업 메뉴로 제공
- 드롭업 내부 항목은 설명 포함 구조를 유지
- 선택 상권 상세 카드는 지도 오버레이로 고정 표시
- 레퍼런스의 분위기 복제가 아니라 정보 구조만 차용
