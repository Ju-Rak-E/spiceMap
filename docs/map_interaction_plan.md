# spiceMap 지도 인터랙션 개선 계획

> 작성일: 2026-04-29  
> 대상: `frontend/src/` (MapLibre GL + Deck.gl)  
> MVP 범위: 강남구 · 관악구 2개 자치구

---

## 1. 상권 클릭 시 관련 흐름 강조

* **목표**: 사용자가 특정 상권을 클릭했을 때, 해당 상권으로 유입되거나 유출되는 OD 흐름만 선명하게 남기고 나머지 흐름은 배경으로 물러나게 한다. 현재는 상권을 클릭해도 전체 흐름이 동일한 강도로 표시되어 "이 상권이 어디와 연결됐는가"를 읽을 수 없다.

* **실현 방법**:
   * **흐름 필터링 — `ODFlowLayer.ts` 수정**: `createODFlowLayer(flows)` 호출 시 `selectedNodeId`를 함께 전달한다. 현재 `Map.tsx`의 `handleFrame` 안에서 `selectedNode?.id`가 이미 CommerceNodeLayer에는 전달되지만 ODFlowLayer에는 전달되지 않는다. `createODFlowLayer(flows, selectedNode?.id ?? null)`로 시그니처를 확장하고, 내부에서 관련 흐름(`originCommCd === selectedId || destCommCd === selectedId`)과 무관 흐름의 알파값을 분기한다.
      * 예: 관련 흐름 알파 → `200` 유지, 무관 흐름 알파 → `20`으로 억제
   * **파티클 레이어 동기화 — `FlowParticleLayer.ts` 동일 적용**: 파티클도 같은 selectedId를 받아 관련 흐름의 파티클만 살린다. 흐름과 파티클이 따로 놀면 오히려 혼란스럽다.
   * **선택 해제 시 전체 복원**: `onClose()` 또는 지도 빈 공간 클릭 시 `onSelectNode(null)`이 호출되면 알파값이 원래대로 돌아온다. 이미 `CommerceDetailPanel`의 `onClose`가 연결되어 있으므로 ODFlowLayer만 맞춰주면 된다.
   * **`Map.tsx` 수정 지점**: `handleFrame` 내부의 `overlayRef.current.setProps`에서 `createODFlowLayer(flows)` → `createODFlowLayer(flows, selectedNode?.id ?? null)`로 교체.

## 2. 영역 표시 및 줌인/줌아웃에 비례한 크기 자동 조절

* **목표**: 사용자가 지도를 확대하거나 축소할 때, 행정동 경계선과 상권 노드가 줌 레벨에 맞게 자연스럽게 변화하도록 한다. 줌 아웃 시에는 자치구 단위 윤곽만 보이고, 줌 인 시에는 상권 단위 세부 경계가 드러나는 구조를 만든다.

* **실현 방법**:
   * **경계선 두께 줌 보간 — `boundaryLayerConfig.ts` 수정**: 현재 `line-width: 0.8`로 고정되어 있다. MapLibre의 `interpolate` 표현식으로 줌 레벨에 따라 선 두께를 동적으로 조절한다.
      * 예: 줌 10 → 0.4px (전체 조망), 줌 13 → 1.2px (상권 탐색), 줌 16 → 2.5px (상세 확대)
      * `'line-width': ['interpolate', ['linear'], ['zoom'], 10, 0.4, 13, 1.2, 16, 2.5]`
   * **행정동 fill 투명도 줌 연동**: 줌 아웃 시 배경 채우기가 너무 강하면 노드가 묻힌다. `fill-opacity`도 줌 레벨에 따라 낮아지도록 설정한다.
      * 예: 줌 10 → opacity 0.15, 줌 14 이상 → opacity 0.25
   * **줌 레벨별 레이어 가시성 전환 — `Map.tsx`에 zoom 상태 추가**: 현재 줌 레벨 상태(`zoomLevel`)를 `useState`로 관리하고, `map.on('zoom')`으로 갱신한다. 줌 11 미만에서는 Deck.gl 노드 레이어를 숨겨 자치구 전체 개요만 보이게 하고, 줌 11 이상에서 상권 노드가 등장하도록 한다.
   * **상권 노드 반경 상수 검증**: `CommerceNodeLayer.ts`의 `MIN_RADIUS = 300`, `MAX_RADIUS = 1500`, `MAX_ABS_FLOW = 1200`은 실제 데이터 기준이 아닌 하드코딩값이다. `od_flows_aggregated` 적재 후 실제 `netFlow` 분포의 90 퍼센타일 값을 `MAX_ABS_FLOW`로 재설정해야 한다. 현재는 목 데이터 기준으로 맞춰져 있다.
   * **상권 경계 폴리곤 (데이터 적재 후 추가)**: `commerce_boundary` 테이블이 적재되면 줌 13 이상에서만 표시되는 폴리곤 레이어를 추가한다. 줌 12 이하에서는 노드만 보이고, 13 이상에서는 해당 상권의 실제 영역 경계가 반투명하게 깔린다. 선택된 상권은 테두리를 강조한다.

## 3. 이동 구간 곡선 표시의 강화

* **목표**: 현재 구현된 2차 베지어 곡선(`flowBezier.ts`)을 기반으로, 이동 방향과 이동량이 한눈에 읽히도록 강화한다. 단순히 곡선이 있는 것과 방향·강도를 시각적으로 전달하는 것은 다르다.

* **실현 방법**:
   * **단거리 흐름 곡률 과잉 문제 수정 — `flowBezier.ts`**: 현재 `CURVE_FACTOR = 0.45`가 거리와 무관하게 고정 적용된다. 강남·관악 내부처럼 행정동 간 거리가 짧은 흐름에서는 곡선이 과도하게 휘어 어색하다. 거리(`dist`)에 반비례하여 곡률을 줄이도록 수정한다.
      * 예: `factor = Math.min(CURVE_FACTOR, 0.15 + dist * 0.6)` — 가까울수록 직선에 가깝게
   * **이동 방향 표현 — `FlowParticleLayer` 활용**: 현재 `PathLayer`는 선만 그린다. 방향은 파티클이 담당해야 한다. 파티클이 출발지에서 도착지 방향으로 흐르는 것이 시각적 방향 단서가 되도록, `FlowParticleLayer`의 진행 방향을 곡선 경로와 일치시키고 파티클 밀도를 이동량에 비례하게 조정한다.
   * **선택된 흐름의 곡선 두께 강조**: 상권 선택 시 관련 흐름의 `width`를 현재 값의 1.5배로 올린다. 같은 색이라도 두께가 달라지면 중요 흐름이 자연스럽게 부각된다.
   * **이동목적 색상 일관성 유지**: 현재 `PURPOSE_COLORS`(출근=하늘, 쇼핑=앰버, 여가=보라, 귀가=초록)는 그대로 유지한다. 색상을 바꾸면 범례와 함께 다시 학습 비용이 생긴다. 강화할 것은 색상이 아니라 선택 상태에서의 대비다.
   * **`MAX_VOLUME = 10000` 재검증**: 이 값이 실제보다 크면 모든 흐름이 최소 두께로만 표시된다. 실제 `trip_count_sum` 분포 확인 후 조정 필요.

## 4. 직관적인 시각적 표시

* **목표**: 지도를 처음 보는 정책 실무자나 심사위원이 5초 이내에 "어떤 상권이 위험하고, 어디서 사람이 오는가"를 읽을 수 있어야 한다. 현재 화면은 색상·크기·움직임이 있으나 그것이 무엇을 의미하는지 즉각 해석되지 않는다.

* **실현 방법**:
   * **노드 크기와 위험도의 분리 명확화**: 현재 노드 크기는 `netFlow(순유입)`를 나타내고, 색상은 `상권 유형(Module D)`을 나타낸다. GRI 위험도는 테두리 색으로만 간접 표현된다. 테두리를 GRI 구간에 따라 명시적으로 구분한다.
      * GRI 70 이상(위험) → 테두리 빨강 `[239, 83, 80]`, 두께 90m
      * GRI 40~69(주의) → 테두리 주황 `[255, 167, 38]`, 두께 60m
      * GRI 39 이하(안정) → 테두리 기존 흰색 계열, 두께 20m
      * 이렇게 하면 색=유형, 크기=유동량, 테두리=위험도로 3가지 정보가 동시에 읽힌다.
   * **상단 종합 해설바 문장 개선 — `summaryFormatter.ts`**: 현재 해설바가 있지만 선택된 상권이 없을 때의 기본 문장이 모호하다. 위험 상권 수를 포함한 문장으로 강화한다.
      * 예: `"강남·관악 상권 127개 · GRI 70 이상 위험 상권 12개 · 출근 이동 상위 30개 흐름 표시 중"`
   * **범례 드롭업 첫 상태 개선 — `CommerceLegend.tsx`**: 현재 기본 상태가 접혀 있다. 처음 접속한 사람은 색상 의미를 모른다. 첫 접속 시 3초간 자동으로 열려있다가 닫히는 동작을 추가한다. 또는 `?` 버튼으로 진입하는 간단한 안내 오버레이를 제공한다.
   * **hover 미니카드 표시 위치 보정**: 현재 `left: x + 14, top: y - 12`로 커서 오른쪽 아래에 고정된다. 화면 오른쪽 끝의 상권을 hover 시 카드가 뷰포트 밖으로 나간다. 화면 우측 경계를 감지해 카드를 왼쪽으로 반전 표시하는 로직이 필요하다.
   * **빈 선택 상태 안내**: 상권을 아직 클릭하지 않은 상태에서 우측 패널 하단이 비어있다. "상권을 클릭하면 상세 분석을 볼 수 있습니다" 같은 안내 문구를 표시해 인터랙션 진입점을 명확히 한다.
