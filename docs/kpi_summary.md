## 1. 가설 검증 (3건)

| ID | 가설 | 결과 (실측) | 임계 | 판정 | 코드 |
|----|------|------------|------|------|------|
| **H1** | net_flow ↑ → sales ↑ | r = **0.106** / p = 2.83×10⁻⁵ / n = 1,565 | r ≥ 0.5, p < 0.05 | **FAIL** (방향성 지지, 효과 약함) | `verification_h1.py` |
| **H2** | barrier ↑ → closure ↑ | r — 산출 대기 (실데이터 실행) | r ≥ 0.3 + p < 0.05 | **PENDING** (함수 구현 완료) | `verification_h2.py` |
| **H3** | Q3 GRI top ↑ → Q4 closure ↑ | gap = **0.746%p** / p ≈ 5×10⁻³⁶ / n top=330, bottom=1,320 | gap ≥ 2.0%p | **FAIL** (방향성 매우 강함, 절대값 미달 — 자치구 매핑 한계) | `verification_h3.py` |

**해석 핵심**: H1/H3 모두 통계적 유의(p ≪ 0.05)를 확보, 방향성은 지지. 절대 효과 크기가 임계 미달인 이유는 **closure_rate가 자치구 단위 데이터를 LATERAL ST_Contains 로 상권에 매핑한 한계 (분산 부족)**. v2 에서 상권 단위 closure 직접 적재 시 개선 예상.
