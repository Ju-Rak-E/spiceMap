# 스파이스 흐름 지도 — 발표용 핵심 기능 5선

> 발표자용 치트시트 · 2026 서울시 빅데이터 활용 경진대회
> 한 줄 요약: **"왜 이 상권이 침체됐는가 — 흐름으로 보는 서울 상권 위험 지도"**

---

## 발표 도입부 (30초 안에 말할 것)

기존 상권 분석은 **매출·폐업률 스냅샷**(결과)만 보여준다.
스파이스 흐름 지도는 **왜 그 상태가 됐는가(원인=흐름)** 를 보여준다.

비유:
> 병원 진단서가 "혈압 180"만 알려준다면 (기존),
> 우리는 "왜 혈압이 올랐는지 — 식습관·운동·스트레스 흐름"을 보여준다 (스파이스).

---

## 기능 1. 흐름 기반 위험 탐지 (Module A·C)

### 한 줄로
매출이 멀쩡해 보여도, **인구 흐름이 끊긴 상권을 미리 잡아낸다.**

### 무엇이 다른가
- 기존: "이번 분기 매출 -5%" → 사후 대응
- 우리: "이 상권에 들어오는 사람이 줄고 있다" → 사전 경고

### 어떻게 동작하나
1. 서울시 OD 데이터(OA-22300, 출발지→도착지 인구 이동)를 **NetworkX 그래프**로 변환
2. 행정동·상권 단위로 **순유입 변화 + 시계열 갭 + 네트워크 중심성** 3가지를 함께 본다
3. 흐름 신호가 약해진 상권을 빨간색으로 표시

### 발표 포인트
> "강남 압구정·청담은 기존 모델에서 '안전'으로 분류됐지만, 흐름으로 보면 침체 징후가 보입니다."

## 기능 2. 젠트리피케이션 위험지수 GRI v1.0 (Module B)

### 한 줄로
**0~100 점수 하나로 젠트리피케이션 위험을 가늠**하는 종합 지표.

### 무엇이 다른가
- 기존: 임대료만, 또는 매출만 따로 봄
- 우리: 4가지 신호를 **하나의 점수**로 결합

### 어떻게 동작하나
4개 항목을 가중 결합:
1. 순유입 증가율
2. 임대료
3. 프랜차이즈 비중
4. 매출 급증률

### 발표 포인트 (★ 가장 강력한 통계)
> "GRI 상위 20% 상권의 다음 분기 폐업률이 하위 80%의 **14.5배**."
> "p값 5×10⁻³⁶, n=1,565 — 우연이 아닙니다."

### UI에서 어떻게 보이나
- **MiniGauge** 컴포넌트: 0~100 게이지로 한눈에
- **TrendChart**: 분기별 추이
- 자치구 내 **백분위 순위 병행 표시** (절대값만 보면 강남이 항상 1등이라 의미 없음)

## 기능 3. 자동 정책 추천 카드 (Module D)

### 한 줄로
상권 상태를 입력하면 **공무원이 바로 결재 라인에 올릴 정책 카드가 나온다.**

### 무엇이 다른가
- 기존: 분석가가 보고서를 손으로 작성 (며칠 소요)
- 우리: 규칙 엔진이 즉시 매칭 (수 초)

### 4가지 정책 룰
| 룰 | 대상 상황 |
|----|----------|
| **R4** | 임대료 급등 → 임대료 안정 정책 |
| **R5** | 프랜차이즈 잠식 → 진입 관리 정책 |
| **R6** | 유동인구 감소 → 공공 유동 유도 정책 |
| **R7** | 폐업 임박 → 긴급 대응 정책 |

### 발표 포인트 (★ 차별점)
> "**생성형 AI를 사용하지 않습니다.** 환각이 없고, 모든 카드에 수치 근거가 명시됩니다."
> "PolicyCard UI에서 **CSV로 즉시 다운로드** → 그대로 결재 문서가 됩니다."

## 기능 4. 실 도로 경로 기반 흐름 단절 시각화

### 한 줄로
**실제 도로 위로 인구 흐름이 흐르는 것**처럼 보이는 애니메이션.

### 무엇이 다른가
- 기존 대시보드: 출발지·도착지 사이를 **직선**으로 연결 (지도 위에 떠 있는 선)
- 우리: **실제 도로망**을 따라 흐름이 흐름

### 어떻게 동작하나
1. **OpenRouteService(ORS) API**로 출발-도착 좌표 사이 실 도로 경로를 받아옴
2. **Deck.gl** 레이어 2종으로 렌더링:
   - `FlowBarrierLayer` — 흐름 단절 구간
   - `DisruptedBarrierParticleLayer` — 단절된 입자 애니메이션

### 발표 시연 안전장치 (★ 중요)
> "ORS API가 발표 중에 죽어도, **자동 fallback + 정적 캐시 재생 모드**로 전환됩니다."
> "발표 끊김 위험 0%."

## 기능 5. 이중 베이스라인 검증 + 시연 모드

### 한 줄로
**"우리 지표가 기존 지표보다 낫다"를 숫자로 증명**하고, 그 증명을 발표 화면에 띄운다.

### 무엇이 다른가
- 기존 서비스: "이게 우리 결과입니다" (검증 과정 비공개)
- 우리: H1·H2·H3 가설 + B1·B3 베이스라인 **5장 카드로 투명 공개**

### 핵심 검증 결과 (★ 발표에서 꼭 인용)

| ID | 무엇을 검증했나 | 결과 |
|----|----------------|------|
| **H1** | 순유입↑ → 매출↑ | r=0.106, p=2.83×10⁻⁵, n=1,565 (방향성 유의) |
| **H3** | GRI 상위 20% → 폐업↑ | **14.5배** 차이, p≈5×10⁻³⁶ (매우 강함) |
| **B1** | 서울시 공식 지표 대비 | **187개 추가 위험 상권 발굴** (공식 미포착) |
| **B3** | 기존 매출 추세 모델 대비 | **231개 추가 신호** (본질적으로 다른 신호) |

### 시연 모드 (Hero Mode)
> URL에 `?hero=1` 붙이면 **키보드 1~4** 로 4구간 자동 시연

| 키 | 화면 |
|----|------|
| `1` | 인트로 — 강남·관악 색 대비 |
| `2` | 신림 골목상권 상세 + R4 정책 카드 |
| `3` | CSV 다운로드 toast |
| `4` | 검증 보고 — H1·H2·H3·B1·B3 5카드 |

## 자주 받을 질문 (Q&A 대비)

**Q. 왜 강남·관악 2개구만?**
> "MVP 범위입니다. 1,650개 상권으로 충분히 통계 유의성(n=1,565)을 확보했고, 서울 전역 확장은 동일 파이프라인으로 가능합니다."

**Q. 생성형 AI 안 쓴 이유?**
> "환각 위험 + 정책은 수치 근거가 명시돼야 결재됩니다. 규칙 기반이면 추적 가능합니다."

**Q. H2가 방향 역전된 이유?**
> "허브 상권 selection bias로 분석했습니다. 검증 보고서에 한계를 명시했습니다 — 숨기지 않습니다."

**Q. 개인정보는?**
> "**개인 이동 추적 없음.** 행정동·상권 단위 집계 데이터만 사용합니다."
