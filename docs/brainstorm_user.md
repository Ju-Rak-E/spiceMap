# 사용자 관점 브레인스토밍 — spiceMap

> flow_map.md 기반 | 2026-04-08

---

## 페르소나 개요

spiceMap의 실제 사용자는 크게 세 유형이다.

| 페르소나 | 역할 | 사용 맥락 |
|---------|------|---------|
| A. 서울시 정책 담당자 | 분기 분석 → 위험 상권 목록 작성 | 사무실, 대형 모니터 |
| B. 자치구 경제과 담당자 | 현장 검토 → 구체 상권 현황 확인 | 사무실 + 현장 이동 중 |
| C. 경진대회 심사위원 | 발표 평가 → 3분 시나리오 관람 | 발표장, 프레젠테이션 화면 |

---

## 페르소나 A — 서울시 정책 담당자

### 프로필
- 서울시 소상공인 지원 또는 도시경제 부서 소속
- GIS, 데이터 분석 기본 소양 있으나 코드는 다루지 않음
- 분기별 보고서 작성 의무, 다수 상권을 한 번에 비교해야 함

### 핵심 사용 시나리오
1. 전체 서울 지도에서 분기별 위험 상권 목록을 확인한다
2. 우선순위 점수 80+ 상권을 클릭해 GRI, 폐업률, 정책 제언을 읽는다
3. 담당 자치구 경제과에 전달할 요약 리포트를 내보낸다

### Pain Points
- 현재 도구(상권변화지표 단독)는 "좋음/나쁨" 이분법적 → 왜 나빠졌는지 설명 부족
- 여러 데이터셋을 수동으로 엑셀에서 합쳐야 하는 반복 작업
- "흐름"이 아닌 "스냅샷"만 보여줌 → 시간 변화 맥락 없음

### 원하는 인터랙션
- **필터**: 자치구, 분기, 상권 유형(흡수형_과열/침체 등)으로 빠르게 좁히기
- **비교**: 두 분기를 나란히 비교하는 타임라인 슬라이더
- **내보내기**: 위험 상권 목록 CSV + 정책 제언 요약 PDF

### 정보 우선순위
```
1순위: 우선순위 점수 + 상권 유형 분류
2순위: GRI 값 + 추세 방향 (악화/개선)
3순위: 정책 제언 (권고 3개 + 담당 부서)
4순위: 원시 데이터 수치 (매출, 폐업률 등)
```

### UX 개선 제안
- 기술 용어 최소화: "net_flow"보다 "순유입량", "centrality"보다 "상권 영향력"
- 색상 인코딩 일관성: 위험(빨강) → 주의(주황) → 안정(초록)으로 직관적
- 정책 제언 카드에 "이 상권에 적용된 사례" 링크 제공 (향후 확장)

## 페르소나 B — 자치구 경제과 담당자

### 프로필
- 특정 자치구 담당 → 전체 서울보다 "우리 구" 중심
- 현장 방문 후 데이터와 현실 차이를 확인하는 경우 많음
- 태블릿/노트북 + 간혹 스마트폰으로 접근

### 핵심 사용 시나리오
1. 담당 구만 필터링해 해당 구 내 상권 흐름을 본다
2. 특정 골목상권 클릭 → 단절 구간 원인(보행 접근? 교통 결절?) 파악
3. 현장 사진과 데이터를 함께 보며 의견서 작성에 활용

### Pain Points
- 서울 전역 지도에서 우리 구를 찾는 것이 불편 (줌인 반복)
- "이 수치가 현장과 다른데 왜?"를 데이터로 설명하기 어려움
- 모바일에서 지도 조작이 느리거나 레이아웃이 깨짐

### 원하는 인터랙션
- **자치구 랜딩**: 진입 시 담당 자치구로 자동 포커스
- **단절 구간 시각화**: 보행 접근 차단, 교통 결절 구간을 지도에서 강조 표시
- **현장 메모 기능**: 특정 상권에 현장 의견 태그 달기 (향후 확장)
- **모바일 반응형**: 패널 접기/펼치기, 큰 터치 타깃

### 정보 우선순위
```
1순위: 우리 구 상권별 유형 분류 + 위험 점수
2순위: 단절 원인 분석 (보행/교통/목적 편향)
3순위: 타 자치구 비교 (우리 구가 얼마나 심각한가)
4순위: 정책 제언 + 담당 부서 연결
```

### UX 개선 제안
- 초기 화면에 "자치구 선택" 온보딩 단계 추가
- 단절 구간을 별도 레이어로 켜고 끌 수 있게 (지도 과부하 방지)
- 오프라인 모드 또는 데이터 캐싱 (현장 이동 중 네트워크 불안정 대비)

## 페르소나 C — 경진대회 심사위원

### 프로필
- 서울시 빅데이터 활용 경진대회 심사 패널 (데이터 전문가 + 정책 전문가 혼합)
- 발표 3분 + 질의응답 제한 → 짧은 시간에 핵심을 파악해야 함
- 여러 팀의 발표를 연속으로 평가 → 임팩트 있는 첫인상이 중요

### 핵심 사용 시나리오 (3분 발표 흐름)
```
0:00 - 문제 제기: 왜 일부 상권만 사람이 몰리는가?
0:30 - 오전 8시 출근 흐름 → 강남·여의도 흡수형 패턴 시각화
1:00 - 오후 2시 쇼핑 흐름 → 홍대·성수 집중 패턴 + GRI 경고
1:30 - 침체 상권 클릭 → 단절 구간 원인 + 정책 제언 카드
2:30 - 분기 전환으로 악화/회복 추세 비교 → "예측 가능한 위험"
3:00 - 결론: 흐름 지도 → 정책 결정 루프 연결
```

### Pain Points (심사위원의 의구심)
- "이 데이터가 신뢰할 수 있는가?" → 출처와 검증 방법이 불분명하면 설득력 저하
- "AI 제언이 근거 없는 생성형 아닌가?" → 환각 우려
- "시각화가 예쁜데 실제로 쓸 수 있나?" → 실용성 증거 필요
- "OD 데이터가 실제 소비를 대변하는가?" → 방법론 취약점 공격 가능성

### 원하는 인터랙션 (발표 중)
- 실시간 인터랙션: 클릭 한 번으로 상권 유형 전환, 애니메이션 매끄러움
- 데이터 출처 표시: 각 지표 옆에 데이터 소스 아이콘 (OA-22300 등)
- 검증 결과 요약: GRI 상관관계 r값, H1~H3 결과를 패널에서 바로 확인

### 설득 포인트 정리
| 심사위원 의구심 | 대응 방안 |
|--------------|---------|
| 데이터 신뢰성 | 공공데이터포털 출처 명시 + 8개 데이터셋 결합 로직 설명 |
| AI 환각 우려 | "규칙 기반 정책 제언, 생성형 AI 아님" 명시 |
| 실용성 의문 | 서울시 정책 연결 프로세스 다이어그램 포함 |
| 방법론 취약점 | H1~H3 검증 결과 + 한계 명시 슬라이드 준비 |

## 공통 UX 원칙 (전체 페르소나)

1. **Progressive Disclosure**: 지도 → 상권 클릭 → 상세 패널 순서로 정보 깊이 조절
2. **Affordance 명확화**: 클릭 가능한 상권 노드는 hover 시 커서 변경 + 하이라이트
3. **컨텍스트 유지**: 필터 변경 시 이전 선택 상권이 유지되어야 함 (맥락 단절 방지)
4. **오류 메시지 친화적**: 데이터 로딩 실패 시 "캐시 데이터로 표시 중" 안내
5. **접근성**: 색각 이상자를 위한 아이콘 보조 표현 (색상만으로 정보 전달 금지)

## 핵심 사용자 질문

1. **"어떤 상권이 가장 위험한가?"** — 우선순위 점수 80+ 목록이 첫 화면에서 보여야 한다
2. **"왜 사람이 안 오는가?"** — 단절 구간 원인이 클릭 한 번으로 펼쳐져야 한다
3. **"지금 악화되고 있는가, 회복 중인가?"** — 분기 타임라인 슬라이더가 직관적이어야 한다
4. **"정책 담당자에게 무슨 말을 해야 하는가?"** — 정책 제언 카드가 복사/내보내기 가능해야 한다
