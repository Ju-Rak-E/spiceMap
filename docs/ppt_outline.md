## 슬라이드 2 — 문제 정의: 기존 도구의 한계

**헤드라인:** 창업자에게 필요한 건 '결과'가 아니라 '원인'입니다

| 기존 도구 | spiceMap |
|----------|----------|
| 매출·폐업률 스냅샷만 제공 | 인구 이동 흐름으로 원인 분석 |
| "이 상권 지금 나쁨" | "왜 나빠졌는지 + 앞으로 어떻게 될지" |
| 추천만 존재 | 추천·주의·비추천 + 각각의 수치 근거 |
| 여러 데이터 수동 취합 | 5개 공공데이터 자동 결합 |

**비유 문구:**
> 병원 진단서가 "혈압 180"만 알려준다면 (기존),
> 우리는 "왜 혈압이 올랐는가 — 식습관·운동·스트레스 흐름"을 보여줍니다.

## 슬라이드 3 — 페르소나

**헤드라인:** 이런 분들을 위해 만들었습니다

### 페르소나 A — 예비 창업자 김지수 (32세)
- 퇴직 후 카페 창업 준비 중
- 강남구 내 어느 골목이 유망한지 모름
- 부동산 중개소 말만 믿기 불안함
- **필요한 것:** "이 상권, 사람이 실제로 얼마나 오가는지 + 비슷한 업종 폐업률"

### 페르소나 B — 소상공인 정책 담당자 박민준 (41세)
- 관악구 경제과 소속
- 분기마다 위험 상권 보고서 수동 작성
- 여러 엑셀 파일 취합에 2~3일 소요
- **필요한 것:** "자치구 내 위험 상권 자동 식별 + 정책 카드 즉시 출력"

**시나리오 한 줄:**
> 김지수가 '카페'를 선택하고 강남구를 클릭하면 — 30초 안에 추천 3곳·주의 3곳·비추천 3곳과 그 이유를 받는다.

## 슬라이드 4 — 서비스 전체 흐름

**헤드라인:** 3단계로 창업 입지를 결정합니다

```
[1단계: 필터 설정]
업종 선택 → 관심 자치구 선택 → 분기 선택

[2단계: AI 분석]
분석하기 버튼 → 5개 지표 가중 합산 → 3등급 분류 → Claude AI 해설 생성

[3단계: 상권 탐색]
추천 카드 클릭 → 지도에서 위치 확인 → GRI 위험도 + 정책 카드 열람
→ OD 흐름으로 시간대별 유동인구 확인
```

*→ 화면 캡처: 전체 UI 구성 (패널 + 지도)*

## 슬라이드 5 — 기능 ①: AI 창업 어드바이저

**헤드라인:** 추천만 주는 게 아닙니다. 왜 비추천인지도 알려드립니다.

**동작 방식:**
1. 업종 + 자치구 선택 후 "분석하기" 클릭
2. 백엔드가 해당 조건의 상권 전체를 5개 지표로 점수화
3. **추천(상위 30%) / 주의(중간 40%) / 비추천(하위 30%)** 3등급으로 분류
4. 각 등급 상위 3개씩 총 9개 선별
5. Claude AI가 수치를 읽고 자연어 해설 생성

**점수 산정 5개 지표:**

| 지표 | 가중치 | 의미 |
|------|--------|------|
| GRI 위험도 | 0.25 | 낮을수록 안전한 상권 |
| 폐업률 | 0.25 | 낮을수록 생존율 높음 |
| 유동인구 (순유입) | 0.20 | 높을수록 고객 접근성 좋음 |
| 업종 점포 수 | 0.20 | 너무 많으면 경쟁 포화 (log 변환) |
| 네트워크 중심성 | 0.10 | 높을수록 상권 연결성 우수 |

*→ 화면 캡처: 추천·주의·비추천 카드 결과 화면*

## 슬라이드 6 — 기능 ②: GRI 젠트리피케이션 위험지수

**헤드라인:** 상권 하나하나에 '위험 온도계'를 달았습니다

**GRI란?**
폐업률 · 인구 순유출 · 네트워크 고립도 3가지를 결합한 0~100 점수.
숫자가 높을수록 해당 상권이 침체·과열 위험에 가까움.

**왜 필요한가?**
- 매출만 보면 강남 압구정은 항상 '안전'으로 분류됨
- 흐름과 연결성을 함께 보면 유입 감소 징후를 **사전에** 포착 가능

**산출 공식:**
```
GRI = 0.40 × z(폐업률) + 0.33 × z(순유출) + 0.27 × z(고립도)
→ percentile rank 0~100 변환
```

*→ 화면 캡처: 상권 클릭 시 GRI 게이지 + 분기별 추이 차트*

## 슬라이드 7 — 기능 ③: OD 이동 흐름 시각화

**헤드라인:** 사람이 어디서 와서 어디로 가는지, 실시간으로 보여줍니다

**데이터 출처:** 수도권 광역 OD 기종점통행량 (공공데이터포털 OA-22300)

**무엇을 볼 수 있나:**
- 출근·귀가·쇼핑·여가 목적별 이동 흐름
- 0~23시 시간대 슬라이더로 시간축 재생
- 상권으로 들어오는 흐름 vs 나가는 흐름 강도 비교

**창업자에게 주는 의미:**
> "오전 10시 이 상권에 쇼핑 목적 유입이 많다 → 카페 운영 시간대 최적화 가능"

*→ 화면 캡처: OD 흐름 애니메이션 켜진 지도 화면*

## 슬라이드 8 — 기능 ④: 자동 정책 추천 카드

**헤드라인:** 상권 분석 결과를 바로 정책 문서로

**대상 사용자:** 소상공인 담당 공무원

**동작 방식:**
상권 GRI 신호에 따라 4가지 룰 중 해당하는 정책 카드를 즉시 생성

| 룰 | 발동 조건 | 권고 정책 |
|----|---------|---------|
| R4 | 임대료 급등 감지 | 임대료 안정화 지원 |
| R5 | 프랜차이즈 잠식 | 업종 진입 관리 |
| R6 | 유동인구 급감 | 공공 유동 유도 사업 |
| R7 | 폐업 임박 신호 | 긴급 소상공인 지원 |

**차별점:** 생성형 AI 없이 수치 기반으로 작동 → 환각 없음, CSV 다운로드로 결재 문서화

*→ 화면 캡처: 정책 카드 + CSV 다운로드 버튼*

## 슬라이드 9 — 데이터 산출 근거 및 AI 활용

**헤드라인:** 3계층 AI가 데이터를 분석합니다

### 계층 1 — 생성형 AI (Anthropic Claude Haiku)
- 역할: 창업 어드바이저 자연어 해설 생성
- 입력: 추천·주의·비추천 9개 상권의 수치 데이터
- 출력: 각 상권별 추천/비추천 이유 한 줄 요약

### 계층 2 — 그래프 네트워크 분석 (NetworkX)
- 역할: 상권 간 이동 네트워크 구축 → 순유입·중심성 계산
- OD 데이터를 행정동 단위 → 상권 단위로 면적 비율 배분

### 계층 3 — 통계 분석 (scipy)
- 역할: GRI 지표의 통계적 유효성 검증
- Pearson 상관 / Welch t-test / z-score 정규화

**데이터 처리 파이프라인:**
```
공공데이터 수집 (Python)
  → PostgreSQL + PostGIS (공간 결합)
  → FastAPI (분석 API)
  → React + Deck.gl (시각화)
  → Claude Haiku (자연어 해설)
```

## 슬라이드 10 — 통계 검증 결과

**헤드라인:** 우리 지표가 실제로 맞는지, 숫자로 증명했습니다

| 검증 | 질문 | 결과 | 의미 |
|------|------|------|------|
| **H1** | 유입↑ → 매출↑? | r=0.106, p=2.83×10⁻⁵, n=1,565 | 방향성 통계 유의 확인 |
| **H3** | GRI 고위험 → 폐업↑? | 14.5배 차이, p=8×10⁻³¹ | GRI 예측력 강하게 입증 |
| **B1** | 기존 공식 지표 대비? | 187개 추가 위험 상권 발굴 | 기존 도구가 놓친 신호 포착 |

**H3 핵심 수치:**
> GRI 상위 20% 상권(330개)의 폐업률이 하위 80%(1,320개)의 **14.5배**
> p = 8×10⁻³¹ — 우연일 확률 사실상 0

**B1 핵심 메시지:**
> 서울시 공식 상권변화지표(OA-15576)가 '정상'으로 분류한 상권 중
> spiceMap이 **187개를 추가로 위험 식별**

*한계도 투명하게 명시: H1 효과 크기 약함(r=0.11), H2 표본 한계(n=39)*

## 슬라이드 13 — 향후 개선 방향 및 확장 가능성
