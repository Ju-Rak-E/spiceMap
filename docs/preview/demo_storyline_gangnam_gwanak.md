# 강남·관악 중심 시연 영상 스토리라인

> 작성: 2026-05-11 · 목적: 3분 내외 시연 영상 녹화용 대본  
> 연결 문서: `docs/preview/hero_shot_scenario.md`, `docs/policy_report_gangnam_apgujeong.md`, `docs/policy_report_gwanak_sillim.md`, `docs/kpi_summary.md`

## 1. 영상의 한 문장

**매출은 결과이고, 사람의 흐름은 원인이다.**

spiceMap은 강남구와 관악구를 단순 비교하지 않는다. 강남은 **흡수형 과열 위험**, 관악은 **방출형 침체 위험**으로 보여주며, 겉보기 매출이 안정적이어도 흐름 변화가 먼저 위험 신호를 보낸다는 점을 설득한다.

## 2. 시청자 관점의 몰입 구조

| 구간 | 시청자가 갖는 질문 | 영상이 주는 답 |
|------|-------------------|----------------|
| 문제 제기 | 왜 또 다른 상권 지도가 필요한가? | 매출이 아니라 흐름의 원인을 본다. |
| 강남 사례 | 좋아 보이는 상권도 위험할 수 있나? | 유입이 강한 상권은 과열·젠트리피케이션 위험이 있다. |
| 관악 사례 | 침체는 매출로 바로 보이지 않나? | 신림은 매출 안정 상태에서도 유입 -38%, 폐업률 +4.2%p가 먼저 나타난다. |
| 정책 연결 | 분석 결과를 실무에 어떻게 쓰나? | 정책 카드와 CSV로 즉시 보고 자료화한다. |
| 검증 | 예쁜 지도 이상의 근거가 있나? | 공식 지표·매출 추세 모델 대비 차별화와 검증 수치를 공개한다. |

## 4. 촬영 컷별 강조 포인트

### 4-1. 오프닝

- 화면 목표: 강남·관악이 한 화면에 잡히고, 색상 대비가 보이게 한다.
- 말의 목표: "상권 상태"가 아니라 "상권 변화의 원인"을 본다는 관점 전환을 먼저 심는다.
- 자막 후보: `매출은 결과, 흐름은 원인`

### 4-2. 강남구 컷

- 화면 목표: 강남 상권 상세 패널에서 GRI와 상권 유형을 보여준다.
- 말의 목표: 강남을 단순 성공 지역으로 설명하지 않고, **흡수형 과열**이라는 다른 리스크로 보여준다.
- 자막 후보: `강남구: 유입 집중 + GRI 상승 = 과열 위험`

### 4-3. 관악구 신림 컷

- 화면 목표: 신림 펄싱, GRI 78, 유입 -38%, 폐업률 +4.2%p가 한 번에 읽히게 한다.
- 말의 목표: "매출은 안정적인데 위험하다"는 반전을 가장 길게 가져간다.
- 자막 후보: `관악구 신림: 매출 안정, 흐름은 이미 위험`

### 4-4. 정책 컷

- 화면 목표: R4 정책 카드와 CSV 다운로드 toast를 반드시 담는다.
- 말의 목표: 분석이 보고서 작성과 정책 개입으로 이어진다는 실용성을 보여준다.
- 자막 후보: `위험 탐지 → 정책 카드 → CSV 보고 자료`

### 4-5. 검증 컷

- 화면 목표: H1·H3·B1·B3 카드가 보이게 한다.
- 말의 목표: 수치의 세부 해석보다 "검증 결과를 숨기지 않고 공개한다"는 신뢰를 준다.
- 자막 후보: `공식 지표 보완 + 매출 모델과 다른 신호`

## 5. 중요 기능과 부가기능 정리

| 분류 | 기능 | 영상에서 보여줄 방식 |
|------|------|---------------------|
| 중요 기능 | 강남·관악 지도 비교 | 첫 화면 자동 포커싱 |
| 중요 기능 | 상권 클릭 상세 패널 | 강남 1회, 신림 1회 클릭 |
| 중요 기능 | GRI 위험지수 | 강남 과열, 신림 침체 모두에서 언급 |
| 중요 기능 | OD 흐름 변화 | 신림 유입 -38% 대사와 함께 표시 |
| 중요 기능 | 정책 추천 카드 | R4 카드 펼침 상태 유지 |
| 중요 기능 | 검증 보고 | 마지막 35초에 신뢰 근거로 제시 |
| 부가기능 | hero mode | `?hero=1`로 시작 |
| 부가기능 | 펄싱 강조 | 신림 컷에서 시선 유도 |
| 부가기능 | CSV export | 정책 컷에서 toast까지 녹화 |
| 부가기능 | 단축키 `1`~`4` | 클릭 실패 시 백업 동선 |

## 6. 짧은 버전 대사

1분 이내 영상이나 발표 예고편이 필요하면 아래 버전을 사용한다.

> "spiceMap은 매출보다 먼저 움직이는 사람의 흐름을 봅니다. 강남은 유입이 강하지만 GRI가 높아 과열 위험이 있고, 관악구 신림은 매출이 안정적인데도 유입이 38% 줄고 폐업률이 4.2%p 올랐습니다. 이 신호는 기존 매출 모델이 놓치기 쉽습니다. spiceMap은 이런 위험을 지도에서 찾고, 정책 카드와 CSV로 바로 실행 가능한 보고 자료까지 연결합니다. 그래서 이 서비스는 예쁜 지도가 아니라, 지자체가 더 빨리 개입하게 만드는 상권 위험 의사결정 도구입니다."

## 7. 녹화 전 체크리스트

- [ ] `https://spice-map.vercel.app/?hero=1` 또는 로컬 `http://localhost:5173/?hero=1` 접속 확인
- [ ] 단축키 `1`~`4` 동작 확인
- [ ] 강남 상권 상세 패널에서 GRI·상권 유형이 보이는지 확인
- [ ] 신림 상세 패널에서 `GRI 78`, `유입 -38%`, `폐업률 +4.2%p`가 읽히는지 확인
- [ ] R4 정책 카드와 CSV 다운로드 toast가 녹화 영역 안에 들어오는지 확인
- [ ] 검증 보고 탭에서 H1·H3·B1·B3 카드가 한 화면에 들어오는지 확인
- [ ] 최종 영상 길이는 2분 50초~3분 10초 안에 맞춘다

## 8. 실패 시 대체 순서

| 문제 | 대체 |
|------|------|
| 클릭이 빗나감 | 단축키 `1`~`4`로 해당 컷 강제 이동 |
| 강남 상권 클릭이 늦어짐 | 강남 컷은 지도 색상 대비와 대사만으로 처리하고 신림 컷 시간을 확보 |
| CSV 다운로드가 브라우저에 가려짐 | toast만 녹화하고 CSV 파일 자체는 보여주지 않음 |
| 검증 카드가 작게 보임 | 브라우저 줌 110% 또는 검증 보고 탭만 별도 컷으로 재녹화 |
| 3분을 초과함 | 강남 컷을 20초, 검증 컷을 25초로 줄이고 신림 컷은 유지 |
